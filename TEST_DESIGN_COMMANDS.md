# 命令行指令测试方案

## 1. 测试概述

本测试方案针对重复文件分析工具的所有命令行指令进行全面测试，确保每个命令的功能正确性和用户体验。

### 1.1 测试目标
- 验证所有命令行指令的功能正确性
- 验证命令参数的有效性
- 验证命令输出的准确性
- 验证错误处理的完整性

### 1.2 测试范围
- index 命令及其子命令
- show 命令及其子命令
- hash 命令及其子命令
- export 命令及其子命令
- config 命令及其子命令
- db 命令及其子命令
- clean 命令及其子命令

## 2. index 命令测试

### 2.1 测试用例：index scan 命令
- **测试目标**：验证扫描路径并建立索引功能
- **测试步骤**：
  1. 创建测试目录和文件
  2. 执行 `index scan <路径>` 命令
  3. 验证输出包含扫描进度信息
  4. 验证数据库中包含扫描的文件
- **预期结果**：扫描成功，索引正确建立

### 2.2 测试用例：index rebuild 命令
- **测试目标**：验证重建重复文件组功能
- **测试步骤**：
  1. 创建测试文件
  2. 执行 `index rebuild` 命令
  3. 验证输出包含重建进度信息
  4. 验证重复文件组正确建立
- **预期结果**：重建成功，重复文件组正确

### 2.3 测试用例：index clear 命令
- **测试目标**：验证清除文件索引功能
- **测试步骤**：
  1. 创建测试文件并建立索引
  2. 执行 `index clear --all --force` 命令
  3. 验证输出包含清除确认信息
  4. 验证数据库中文件索引已清除
- **预期结果**：清除成功，文件索引已清空
